1.表引擎的选择：

1.1 MYISAM  支持全文索引 表级索
1.2 INNODB  支持事务 行级索
1.3 MEMORY  储存在内存当中

2.字段数据类型：

2.1 整数：int(最长10位)、tinyint(最长3位) 

2.2 字符串：char（定长）、varchar(变长),text(文本：字段设计时 不能设置默认值 不能设置长度 新增不插入值时默认为null);  字符集为 uft8 排序规则为：utf8_general_ci

2.3 日期时间：int(可存时间戳 默认值为：0)、date (2019-07-15 默认值为：'0000-00-00')、datetime (2019-07-15 15:25:30 默认值为：'0000-00-00 00:00:00')

2.4 小数：decimal(10,2)  总长度为10,整数8位,小数2位.

2.5 enum(枚举型)：只能从枚举的列表中选择一个 当插入一个不在枚举列表(索引从1开始 例如 1,2,3;空字符串的索引为0,null值的索引为null)中的值时 会插入一个空字符串（''）;  ENUM 被声明为 NULL，NULL 也是该列的一个合法值，并且该列的缺省值也将为 NULL 。如果一个 ENUM 被声明为 NOT NULL，该列的缺省值将是该列表所允许值的第一个成员

2.6 set(集合)：能从枚举的列表中选择多个   当插入一个不在枚举列表(索引为等比2的数列 例如 1,2,4,8;空字符串的索引为0,null值的索引为null)中的值时 会插入一个空字符串（''）   set 被声明为 NULL，NULL 也是该列的一个合法值，并且该列的缺省值也将为 NULL 。如果一个 set 被声明为 NOT NULL，  该列的缺省值将是该列表所允许值的第一个成员

3 字段属性

3.1 UNSIGNED （无符号） 修饰数字类型的字段

3.2 AUTO_INCREMENT 修饰自增长的字段 一般为主键字段 不能设置默认值

3.3 null 向数据库插入值时 该字段可以不赋值 默认为 NULL值 不会报错。

3.4 not null 向数据库插入值 该字段不赋值时 会报错。若设置了默认值 default  不赋值时 则值为该默认值，不会报错  在表设计时 尽量使用 not null,不要使用null

3.5 default 当字段属性中包含not null时 为该字段设置默认值 default '值'

3.6 comment  设置字段或表的注释值  comment '值'
