绝对路径：路径的写法“一定由根目录 / 写起”，例如： /usr/share/doc 这个目录。

相对路径：路径的写法“不是由 / 写起”../man 或 ../../man 或 ./man


目录的相关操作命令

1.cd （ change directory, 变换目录）

. 代表此层目录

.. 代表上一层目录

- 代表前一个工作目录

~ 代表“目前使用者身份”所在的主文件夹

~account 代表 account 这个使用者的主文件夹（ account是个帐号名称）

2.pwd：显示目前的目录

-P ：显示出确实的路径，而非使用链接 （ link） 路径。

3.mkdir：创建一个新的目录

-m ：设置文件的权限喔！直接设置，不需要看默认权限 （ umask） 的脸色～
-p ：帮助你直接将所需要的目录（包含下层目录）创建目录树
例：mkdir user/{xx yy zz} 相当于创建user目录时，也同时在user目录下面同时创建 xx、yy、zz三个子目录

4.rmdir：删除一个空的目录

-p ：递归删除空目录， 连同“下层”“空的”目录也一起删除

2.cp （ 复制文件或目录）

-a ：相当于 -dr --preserve=all 的意思，至于 dr 请参考下列说明；（ 常用）
-d ：若来源文件为链接文件的属性（ link file） ，则复制链接文件属性而非文件本身；
-r ：递回持续复制，用于目录的复制行为；（ 常用）
-p ：连同文件的属性（ 权限、用户、时间） 一起复制过去，而非使用默认属性（ 备份常用）；
-i ：若目标文件（ destination） 已经存在时，在覆盖时会先询问动作的进行（ 常用）


-a 

-p

3.rm （ 移除文件或目录）

-rf  强制递归移除文件或目录

4.mv （ 移动文件与目录，或更名）

-i ：若目标文件 （ destination） 已经存在时，就会询问是否覆盖！

文件内容查阅命令

1.cat 由第一行开始显示文件内容

2.tac 从最后一行开始显示，可以看出 tac 是 cat 的倒着写！

3.nl 显示的时候，顺道输出行号！

4.more 一页一页的显示文件内容

5.less 与 more 类似，但是比 more 更好的是，他可以往前翻页！以下是使用方法：

-Ng
-g：不加亮显示搜索到的所有关键词，仅显示当前显示的关键字，以提高显示速度； 
-N：每一行行首显示行号；

[pagedown]：向下翻动一页；
[pageup] ：向上翻动一页；
[ Enter]:逐行向下显示；
/字串 ：向下搜寻“字串”的功能；
?字串 ：向上搜寻“字串”的功能；
n ：重复前一个搜寻 （ 与 / 或 ? 有关！）
N ：反向的重复前一个搜寻 （ 与 / 或 ? 有关！）
g ：前进到这个数据的第一行去；
G ：前进到这个数据的最后一行去 （ 注意大小写） ；
q ：离开 less 这个程序；

6.head 只看头几行

7.tail 只看尾巴几行

8.od 以二进制的方式读取文件内容！

修改文件时间或创建新文件
1.touch （创建一个空的文件；将某个文件日期修订为目前 （ mtime 与 atime））

指令与文件的搜寻
1.which (搜寻可执行文件-即命令)
-a  将所有由 PATH 目录中可以找到的指令均列出，而不止第一个被找到的指令名称
例：which cd

2.whereis ( 由一些特定的目录中寻找文件文件名)
-l :可以列出 whereis 会去查询的几个主要目录而已
-b :只找 binary 格式的文件
-m :只找在说明文档 manual 路径下的文件
-s :只找 source 来源文件
-u :搜寻不在上述三个项目当中的其他特殊文件

3.locate：依据 /var/lib/mlocate 内的数据库记载，找出使用者输入的关键字文件名 如果你忘记某个文件的完整文件名时 使用此命令比较好
4.updatedb：根据 /etc/updatedb.conf 的设置去搜寻系统硬盘内的文件名，并更新/var/lib/mlocate 内的数据库文件；

5.  find [PATH] [option] [action]
例子：find /home -name  1.txt
